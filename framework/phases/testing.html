Phase 3: Testing & Production

Testing happens in three waves

Testing Waves

Progressive testing from focused groups to production launch.

3.1

Initial Testing

Agent Owner Implementation Team

Initial Testing puts the agent in front of real users for the first time—but in a controlled way.

A small, focused group tests the agent. These are typically the "first users" identified in the Roll Out Strategy Workstream: supportive, representative, and willing to provide candid feedback.

The goal is fast feedback and rapid iteration. Issues discovered here are fixed quickly while the user base is small. The agent's personality, tone, and behavior are fine-tuned based on how real users actually interact with it.

The Agent Owner gathers feedback systematically and works with the Implementation Team to address issues. Speed matters—this phase should move quickly to maintain momentum and tester engagement.

3.2

Broader Testing

Agent Owner Business Team

Broader Testing expands the user base to cover different roles and personas identified during Parallel Exploration.

This phase reveals what people really want versus what was assumed. Different user types interact with the agent differently. Some find it intuitive; others struggle. Some use it for expected purposes; others discover unexpected applications.

The Agent Owner and Business Team work together to understand these patterns. What works? What doesn't? Where are the gaps between design assumptions and actual needs?

Findings from this phase inform final refinements before production. Some issues may require fixes; others may become known limitations to address post-launch.

3.3

Production Launch

Agent Owner Business Owner

Production Launch is when the agent becomes real. It moves from testing to actual use by the intended user base.

Launch happens when core functionalities are satisfactory. This doesn't mean perfect—early launch is expected and encouraged. An agent that works well enough to generate real data will improve faster than an agent stuck in endless testing.

The Business Owner makes the go/no-go decision. This is a gate: Does the agent meet the minimum criteria established in the Evaluation Workstream? Is it ready for production?

Once launched, the agent enters the continuous loop of Improvement and Adoption. Real data drives improvement. The Agent Owner's focus shifts from building to evolving.
